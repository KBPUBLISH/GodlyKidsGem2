================================================================================
GEMINI 3 WORKFLOW - COMMANDS TO RUN ON YOUR LOCAL MACHINE
================================================================================

⚠️ CRITICAL: Gemini 3 cannot access GitHub!
You MUST pull latest code FIRST, then give Gemini 3 the UPDATED code.
Otherwise, Gemini 3 will work with outdated code!

================================================================================
BEFORE GEMINI 3 (CRITICAL - DO THIS FIRST!):
================================================================================

1. Open Terminal/Command Prompt on your computer
2. Navigate to project:
   cd /path/to/GodlyKidsGeminiProject

3. Pull latest changes from GitHub:
   git pull origin main

4. Verify you have latest code:
   git log --oneline -5

5. Open the file you want Gemini 3 to modify
6. Copy the ENTIRE file content (this is the updated version)
7. Paste it into Gemini 3 prompt

IMPORTANT: Always mention "I just pulled the latest code" in your Gemini 3
prompt so you remember you're giving it current code.

================================================================================
AFTER GEMINI 3 PROVIDES MODIFIED CODE:
================================================================================

1. Copy the modified code from Gemini 3
2. Save it to the file in your project (overwrite the old file)
3. Test if possible

4. In Terminal, run these commands:
   git add .
   git commit -m "Description of what Gemini 3 changed"
   git pull origin main
   git push origin main

================================================================================
QUICK REFERENCE - COPY AND PASTE THESE:
================================================================================

# BEFORE Gemini 3 (CRITICAL!)
git pull origin main
git log --oneline -5

# AFTER Gemini 3 (save files first)
git add .
git commit -m "Your message here"
git pull origin main
git push origin main

# Check if you're up to date
git status

# See what changed
git diff

# If you have uncommitted work and need to pull
git stash
git pull origin main
git stash pop

================================================================================
EXAMPLE WORKFLOW:
================================================================================

1. In Terminal: git pull origin main
2. In Terminal: git log --oneline -5  (verify latest)
3. Open App.tsx, copy entire content
4. In Gemini 3: "I just pulled the latest code. Here's App.tsx: [paste]"
5. Copy modified code from Gemini 3
6. Save to App.tsx in your project
7. In Terminal:
   git add App.tsx
   git commit -m "Updated App.tsx from Gemini 3"
   git pull origin main
   git push origin main

================================================================================
SYNCING BETWEEN CURSOR AND GEMINI 3:
================================================================================

If you made changes in Cursor:
1. In Cursor terminal: npm run push (or git push)
2. Then in your Terminal: git pull origin main
3. Open file, copy updated content
4. Give to Gemini 3

If you made changes with Gemini 3:
1. Save files, then in Terminal: git add . && git commit -m "msg" && git push
2. Then in Cursor: npm run sync (or git pull)

================================================================================
COMMON MISTAKES TO AVOID:
================================================================================

❌ Giving Gemini 3 old code
✅ Always pull first, then give updated code

❌ Forgetting to pull before Gemini 3
✅ Make it a habit: Pull → Open → Give to Gemini 3

❌ Not checking if code is up to date
✅ Use git status and git log to verify
